Explications techniques

Notre architecture se compose comme ceci :

  • Le jeu Unity est un client lourd qui sert d'interface pour communiquer avec les chatbots
  • Le serveur NGINX embarqué dans une image Docker sert de point d'entrée pour l'API écrite avec Flask
  • L'application Flask embarqué dans une image Docker sert à distribuer les requêtes vers les chatbots visés
  • Les chatbots entraînés grâce à la librairie open-source Rasa se chargent de répondre aux questions posées

Défi ProgBot

Lien du défi ici !

Nous avons traité ce défi avec celui nommé "Intelligence Artificielle", les explications pour ces deux défis sont donc exactement les mêmes.

La partie IA sera présenté dans cette section. Il vous est recommandé d’avoir lue les explication concernant l’objectif général du projet afin de comprendre aux mieux le choix fait. Dans un premier nous présenterons comment on été réaliser les chatbots dans le jeux. Puis dans un second temps un chatbot très spéciale a été ajouté aux près du bot d'accueil.

Comme vous avez pue lire l’étudiant est amené à rencontrer différent PNJ (personnage non joueur) qui en réalité sont des bot tousse différent. L’objectif est que les entreprise tel que le Crous, CAF, Les universités mette à disposition des outils du type chatbot pour automatiser aux plusse les démarche longue est difficile. Mais la question est comment leur apporter une méthode haut niveaux pour réaliser un chatbot. Pour se faire nous avons utilisé la librairie RASA. Il s'agit d’un projet open-source python pour réaliser des chatbot de type domain fermer. La construction d’un bot avec Rasa est très similaire a celle de dialogflow. Cette librairie combine plusieur technique différente de machine learning. Par exemple du deep learning pour la prédiction des actions a faire en fonction de l’intention détecter On retrouve 2 notion importante les intentions (ce que souhaite faire l’utilisateur) les entitée (les informations a extraire dans le text taper par l’utilisateur). Les avantages de cette librairie est la faciliter pour réaliser un chatbot.

L'inconvénient de Rasa c'est qu’il s'agit d’un bot du type domain fermer. Il faut donc créer des scénarios. Ajouter des phrases pour qu'une intention soit bien détecté, etc.

Pourquoi le choix de Rasa est qu’il s'agit d’un librairie open-source est celui-ci n’est pas hébergé auprès de grand entreprise qui pourrait récolter certaine donnée sensible.

Pour l’avenir de ce projet chatbot. On peut imaginer de créer une interface plus haut niveaux afin de permettre a n’importe qui de développer son chatbot pour le jeux. Ou encore aider la génération de phrase est détection d'entités afin de rendre la démarche de création moins difficile. L’objectif du chatbot info est de attribuer des quêtes à l’étudiant en fonction de ça question. Pour une question de temps que une seul quête a été implémenté la recherche d’un logement aux crous. L’or ce que vous aller communiquer avec celui-ci vous aller donner votre nom prénom, email. Afin que tout les autre bot vous connaisse. Puis si vous luis poser une question du type need a room. Il vous donneras la quête pour trouver un logement aux crous.

Mais ce Bot est très spécial il est doté d’un second réseau de neurone qui peut mener une conversation diverse aux lieu de demander un logement aux crous vous pouvez luis dire des phrase du type “I love pasta and you?”. Une explication sera donnée dans la second partie.

Nous allons vous présenter comment fût réalisé le chatbot du type domaine ouvert. L’objectif est d'utiliser le nouveau modèle entrainé par OpenAi GPT2 afin d’appliquer un transfer learning sur notre modèle open bot. Nous nous sommes aidés de ce projet.

Nous l’avons intégré avec le chat bot info. L’idée ici est de montrer la possibilité de rajouter un autre réseau de neurone sur un chatbot rasa. Ce qui permet d'illustrer la possibilité de rendre un bot beaucoup plus intelligent en combinant simplement différentes architectures de deep learning. Voici une vidéo du open bot :



Les deux dernières vidéos montrent les deux chatbots s'exécuter en individuel (crous, info). Afin de montrer leur comportement. Car nous avons eu quelque problème de communication avec le jeux est le serveur ce qui cause une défaillance dans le fonctionnement de la conversation dans le jeux.

Comme vous pouvez voire dans la première vidéo l’étudiant n’est pas connus du système donc le bot info luis demande un minimum d’information pour que les autres chatbots puisse fonctionner. Avec le deuxième vidéo que le bot crous est capable de reconnaître l’utilisateur.

L’idée recherché est de centraliser les information de l’étudiant aux prêt de tout les organisme (Crous, CAF, etc)

Note: les /start /crous_get_files, sont des commande exécuter par le jeux mais pour la démonstration manuel nous devons les exécuter.


Pour l’architecture internet ChatBot nous avons 2 instances RASA qui sont éxecuter. Celui pour le bot Crous celui pour le bot info. Puis notre bot open-bot (domaine ouvert) qui communique avec le bot info. Pour faciliter l’extraction des mail est déchiffré nous avons ajouté un server Duckling. Lorsque l’étudiant entre un texte. Rasa l’envoie a Duckling pour extraire les potentielles entités. Le tout peut être accessible à l’aide de notre api publique.

Pour communiquer avec le bot info. Il faut faire une requête post à cette addresse .

Avec ce type de contenu json: {"user_id": "toto", "message": "hi", "bot_id": "5005"}

Si vous voulez amorcer la conversation avec un bot il faut entrer "/start"

Pour le bot crous:

Ce serious game a pour but de sensibiliser l'étudiant à l'éventail des démarches administratives et des aides qui lui sont disponibles.

Projeté dans un univers en 2 dimensions, le joueur va devoir parcourir le monde en quête d'informations sur ses démarches étudiantes. Tout au long de son périple, il va parler à différents personnages afin d'atteindre ses objectifs. Chaque personnage détient des informations particulières et possède une personnalité propre, développée grâce aux dernières techniques d'intelligence artificielle sur les chatbots.

Pour développer ce jeu, nous avons utilisé le moteur de jeu vidéo Unity dans le but d'effectuer un prototype rapide de notre application. Celui-ci va communiquer en JSON aux chatbots via une api rest developpée en flask. L'api est chargée de répondre aux requêtes de l'utilisateur, sachant que à chaque png correspond un chatbot. Chaque chatbot se présente sous la forme d'un micro-service.

Dans un avenir proche, le jeu se verra doté de nouvelles fonctionnalités, comme par exemple l'ajout d'une dimension communautaire et multijoueurs avec des espaces d'échanges, de commentaires, etc... Des liens avec les organisations réelles permettront aux joueurs d'effectuer directement leurs démarches à partir du jeu. Enfin, on peut imaginer l'ajout d'options d'accessibilité via des contrôles vocaux, ou encore via les graphismes adaptés pour le jeu.
